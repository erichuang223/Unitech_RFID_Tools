# 2026-06-13 00:25 — 工具核心需求：盤點觸發測試 + 光棒選單 UI

## 使用者明確要求（原意鎖定）

1. **工具必須能實測「讀取觸發機制」**——使用者目前不確定設備是「下指令才讀一次」還是「開關打開後一直收」，
   而應用端要在工廠現場收集 RFID，怎麼收與現場環境高度相關。因此工具要把以下做成**可實測的一級功能**：
   - **單一觸發收集**（single trigger collect）
   - **開啟連續收集**（start continuous collect）
   - **關閉連續收集**（stop continuous collect）
   - 並能看到即時標籤清單、計數、原始封包，以親眼驗證實機真實行為。

2. **選單式 Console UI（光棒導覽）**——不是打字 REPL，而是：
   - 選單階層順序：**分類（Category）> 功能 / 設定（Function / Setting）**。
   - 以 **Console 光棒（HighLine / highlight bar）** 方式選取：方向鍵移動光棒、Enter 進入分類或開啟功能。

## 我對「觸發機制」的分析（供規格採用，仍需實機驗證）

依指令集，RM300P 採「下指令觸發 + 非同步串流回傳」，非硬體 On/Off：
- 觸發：Tag Inventory `0x80`(3.1) → 標籤資料以 `0xC0`(4.1) 串流回。
- 單次/連續：Operation Mode `0x116/0x117`(5.21/5.22)，OPMODE 0=連續、1=非連續。
- 結束通知：Inventory Status `0xC7`(4.8) INVSTATUS=0 Done。
- 停止：Cancel `0x81`(3.2)。
- 對應：單次=非連續模式送 0x80 收到 Done；連續=連續模式送 0x80 持續收；關閉=送 0x81。
- 註：規格行為仍以實機實測為準，這正是工具存在的理由。

## 影響

- 第一版 Console UI 改為「光棒選單導覽」（取代先前的打字 REPL 構想）。
- 「盤點收集」為一級分類，內含單次/連續開/連續關/即時檢視。
