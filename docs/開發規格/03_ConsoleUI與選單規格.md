# 開發規格書 03 — Console UI 與選單規格

> bench tool 開發規格書第 03 篇。定義 Console 前端的**光棒選單互動模型**、選單樹與畫面佈局。
> 依賴 [02 指令層](02_指令層規格.md) 的 API。UI 與核心庫分離原則見 [00 總覽](00_總覽與架構.md)。

---

## 1. 互動模型：光棒（Highlight Bar）選單導覽

**非打字 REPL，而是方向鍵選單。**

| 按鍵 | 動作 |
|---|---|
| ↑ / ↓ | 上下移動光棒（反白）到不同的選單項目 |
| Enter | 進入該分類（下一層選單）或執行該功能 |
| Esc / ← | 返回上一層選單 |
| Home / End | 跳到第一項 / 最後一項（選配） |
| 數字鍵 | 快速跳到對應序號項目（選配） |

- 光棒項目以**反白（背景色）**標示目前選取項；其餘正常顯示。
- 選單階層順序固定為：**分類（Category）> 功能 / 設定（Function / Setting）**，必要時功能下再帶參數輸入。
- 實作方式：**Spectre.Console 套件**（2026-06-16 拍板）；互動行為以本篇為準。

## 2. 畫面佈局

畫面分三區，常駐顯示連線狀態與最近一次收發：

```
┌──────────────────────────────────────────────────────────┐
│ RM300P Bench Tool   [連線: COM3 @115200  ●已連線]          │  ← 標題列（含連線狀態）
├──────────────────────────────────────────────────────────┤
│  ▶ 盤點收集                                                 │
│    裝置資訊                                                 │  ← 選單區（光棒反白「▶」當前項）
│    標籤讀寫                                                 │
│    參數設定                                                 │
│    事件通知                                                 │
│    診斷監看                                                 │
│    連線                                                     │
├──────────────────────────────────────────────────────────┤
│ [結果] STATUS=0x00 成功；韌體版本 = 0.0.0.1                  │  ← 結果/解析區（中文）
│ [收] 02 00 09 00 21 00 00 00 00 01 FF D5 03                 │  ← 原始 hex 監看（收/送）
│ [送] 02 00 04 00 21 FF DB 03                                │
└──────────────────────────────────────────────────────────┘
 ↑↓ 移動   Enter 選擇   Esc 返回                               ← 操作提示列
```

- **標題列**：工具名 + 連線狀態（埠名、baud、連線中/斷線，以顏色/符號區分）。
- **選單區**：當前層級的項目清單，光棒反白當前項。
- **結果/解析區**：最近一次指令的**中文解析結果**（成功/失敗、解出的資料、錯誤碼中文）。
- **hex 監看**：最近收/送的原始位元組（可於「診斷監看」開完整捲動視窗）。
- **操作提示列**：當前可用按鍵。

## 3. 選單樹（分類 > 功能 / 設定）

> 範圍內項目；已排除 Kill / Lock / BlockPermalock / 韌體更新（見 [00](00_總覽與架構.md)、[05](05_安全非功能與驗收.md)）。

```
[主選單 — 分類]
├─ 連線 (Connection)
│   ├─ 選擇 COM Port            （列出可用埠，光棒選取）
│   ├─ 設定 Baud Rate           （115200 / 460800 / 921600 / 自訂）
│   ├─ 連線 / 斷線
│   └─ 連線狀態
│
├─ 盤點收集 (Inventory)         ★核心
│   ├─ 盤點模式設定             （連續 / 非連續，對應 0x116/0x117）
│   ├─ 單次盤點收集             （非連續 → Start → 收到 Done 自動停）
│   ├─ 開啟連續收集             （連續 → Start → 串流）
│   ├─ 停止連續收集             （Cancel 0x81）
│   └─ 即時標籤清單檢視         （見 §4）
│
├─ 裝置資訊 (Device Info)       唯讀
│   ├─ 韌體版本 / 機型 / 序號 / SKU
│   ├─ 環境溫度 / PA 溫度
│   └─ 重置裝置                 （二次確認；恢復原廠再次確認）
│
├─ 標籤讀寫 (Tag R/W)
│   ├─ 讀取標籤                 （輸入 ACCPW/Bank/Addr/Len）
│   └─ 寫入標籤                 （限測試標籤；二次確認）
│
├─ 參數設定 (Configuration)
│   ├─ 區域 Region
│   ├─ 天線（功率 / 狀態 / 盤點輪數）
│   ├─ RF Mode
│   ├─ Gen2（演算法 / 查詢群組）
│   ├─ LBT（開關 / 門檻）
│   ├─ FastID / TagFocus / 相位輸出
│   ├─ 溫度保護（門檻 / 模式）
│   ├─ 頻率 / 發射時間
│   ├─ GPIO（設定 / 狀態）
│   ├─ 系統時間 / 省電
│   ├─ UART 鮑率           （改錯會失聯，明顯警告）
│   └─ 設定儲存模式        （測試期建議 RAM-only）
│
├─ 事件通知 (Events)
│   ├─ 啟用 / 停用 天線狀態事件 (0xE0)
│   └─ 啟用 / 停用 LBT 狀態事件 (0xE1)
│
└─ 診斷監看 (Diagnostics)
    ├─ 原始 hex 收發監看（完整捲動）
    └─ 操作日誌（含時間戳）
```

## 4. 盤點即時檢視畫面 ★核心

進入「即時標籤清單檢視」或啟動收集後，畫面以**即時刷新表格**呈現：

```
盤點中… 模式: 連續    已讀: 1234 筆 / 唯一標籤: 57    [Esc 停止並返回]
┌─────┬──────────────────────────────┬──────┬──────┬───────┐
│ ANT │ EPC                          │ RSSI │ 次數 │ 末次  │
├─────┼──────────────────────────────┼──────┼──────┼───────┤
│  1  │ AAAA3412DC03011716 10B988    │ -52  │  37  │ 00:12 │
│  2  │ E20034120138010006 7A8E7F    │ -61  │  20  │ 00:12 │
└─────┴──────────────────────────────┴──────┴──────┴───────┘
```

規格要點：
- 表格欄位：天線、EPC（去重彙整）、最新 RSSI、累積讀取次數、最後一次讀到時間。
- 標頭顯示：**目前模式（連續/非連續）**、累積讀取筆數、唯一標籤數、運作狀態。
- **連續模式**：表格持續刷新，直到使用者按 Esc（觸發 Cancel）。
- **單次模式**：盤完一輪（收到 `InventoryDone` 0xC7）自動停止並顯示「本輪完成」。
- 由 `TagReported` / `InventoryDone` 事件驅動更新（見 [02](02_指令層規格.md) §3、§4）。
- 即時檢視期間，畫面下方仍可看到原始封包計數（完整 hex 在診斷監看）。

## 5. 功能操作流程通則

- **唯讀功能**（裝置資訊）：選取即執行 → 結果區顯示中文解析。
- **設定功能**（參數設定）：採 **Get → 顯示目前值 → 讓使用者輸入新值 → Set → 再 Get 驗證** 流程。
- **需參數的功能**（讀寫、天線設定等）：以逐欄輸入提示（含預設值/範圍說明）收集參數後執行。
- **危險功能**（重置、寫入、恢復原廠）：執行前**二次確認**（見 [05 安全](05_安全非功能與驗收.md)）。
- **任何失敗**：結果區以紅色/明顯標示，顯示 STATUS 或錯誤碼的**中文說明**，並保留原始 hex 供排查。

---

## 6. 非設備狀態下的行為

- 未連線時，需要設備的功能應變灰/提示「請先連線」，不送出指令。
- 指令逾時（見 [01](01_傳輸與協定層規格.md) §1.6）：結果區顯示「逾時，請確認連線/baud」。
- 收到非預期或損毀封包：記錄到日誌，不中斷選單操作。
