# 開發規格書 05 — 安全規範、非功能需求與驗收

> bench tool 開發規格書第 05 篇（末篇）。定義安全/防呆規範、非功能需求、階段驗收標準與未決事項。
> 範圍界定見 [00 總覽](00_總覽與架構.md)。

---

## 1. 安全與破壞性操作規範

### 1.1 第一版排除清單（不實作）

| 操作 | 指令 | 原因 |
|---|---|---|
| Tag Kill | 0x85（[3.6](../Chapter3_RFIDOperation.md)） | 標籤永久報廢，不可逆 |
| Tag Lock | 0x84（[3.5](../Chapter3_RFIDOperation.md)） | 含永久鎖位元，風險高 |
| Tag BlockPermalock | 0x87（[3.8](../Chapter3_RFIDOperation.md)） | 記憶體永久鎖定，不可逆 |
| 韌體更新 | 8.1~8.3（[第 8 章](../Chapter8_FirmwareUpdate.md)） | 失敗可能變磚 |

- 這些指令的**文件保留**（第 3、8 章不刪），協定/指令層可預留位置，但 UI 第一版**不提供入口**。
- 若未來要納入，必須先補上專屬防呆流程（明確警語、輸入確認碼、犧牲標籤等）。

### 1.2 範圍內的危險動作防呆

| 動作 | 防呆要求 |
|---|---|
| 重置裝置 (0x2F) | 二次確認；選「恢復原廠」時再次明確確認 |
| 寫入標籤 (0x83) | 二次確認；明示「僅用測試標籤」；顯示將寫入的 Bank/Addr/資料 |
| 設定 UART 鮑率 (0x151) | 明顯警告「改錯將導致失聯、需重開機生效」，並提示記住新值 |
| 任何 Set 設定 | 測試期預設將「設定儲存模式」設為 **RAM-only (0x1F1)**，避免誤寫 Flash；需要永久化時由使用者明確切回 |

### 1.3 預設安全姿態

- 連線後**不自動送出任何會改變狀態的指令**；唯讀查詢才可選擇性自動執行。
- 未連線時，需要設備的功能不可送指令（見 [03 §6](03_ConsoleUI與選單規格.md)）。

## 2. 非功能需求

| 類別 | 需求 |
|---|---|
| **日誌** | 所有收發封包（含時間戳、解析結果、錯誤）可記錄到檔；診斷監看可即時檢視 |
| **效能** | 連續盤點不掉包、UI 不卡頓；能呈現約略讀取速率（設備規格上限 >600 tags/s） |
| **穩定性** | 長時間連續收集無記憶體洩漏、不崩潰；斷線能偵測並提示重連 |
| **回應性** | 背景讀取與 UI 分離，光棒操作不被序列 I/O 阻塞（見 [01 §1.4](01_傳輸與協定層規格.md)） |
| **錯誤處理** | 逾時、損毀封包、非預期回包皆有明確處理，不使程式中止 |
| **可測試性** | CHECKSUM/編碼/分包/解析可離線單元測試（見 [04 階段 -1](04_確認測試規格.md)） |
| **可維護性** | 核心庫與 UI 分離；指令對應集中管理，易於擴充新指令 |
| **本地化** | 結果與錯誤一律提供**繁體中文**說明（對應各章中英對照表） |

## 3. 驗收標準

### 3.1 階段 2（本規格書）結案門檻

- 六份規格文件齊備、互相引用一致、可正確跳轉。
- 已涵蓋：架構、傳輸/協定（含已驗證的 CHECKSUM 與分包定義）、指令對應、Console 光棒 UI 與選單樹、
  確認測試矩陣（含盤點三模式）、安全與非功能。
- **使用者審閱通過**。

### 3.2 階段 3（實作）完成定義（前瞻，供參考）

- 依本規格書產出 `RM300P.Core`（傳輸+協定+指令）與 `RM300P.ConsoleTool`（光棒選單）。
- 通過 [04](04_確認測試規格.md) 的階段 -1 全部單元測試。
- 在實機上通過階段 0~1（連線 + 握手），並能執行盤點三模式。

### 3.3 階段 4/5（測試/驗收）前瞻

- 以 [04](04_確認測試規格.md) 階段 2~6 對實機完成確認測試，產出測試結果記錄。
- 完成階段 4.4「實機盤點觸發行為結論」，作為應用端現場收集策略依據。

## 4. 未決事項與假設

| # | 項目 | 現況 / 假設 |
|---|---|---|
| 1 | RM300P 預設 baud rate 與接腳 | 假設 115200/8N1；最終以原廠 RM300P 硬體規格書為準 |
| 2 | 光棒選單實作方式 | Spectre.Console vs 手刻 `Console.ReadKey`，階段 3 決定 |
| 3 | 實機盤點觸發真實行為 | 規格推論為「指令觸發 + 模式控制連續/單次」，**待階段 4 實測確認** |
| 4 | 流控（CTS/RTS） | 第一版預設不用；若實測掉包再評估 |
| 5 | 舊檔 `docs/Unitech_RFID_Command_Set_V1.02.md` | 保留/刪除仍待使用者決定（自階段 1 延續） |
| 6 | 自訂盤點格式 (0xA0/0x128/0x129) 的完整子格式 | 第一版以基本盤點為主，自訂格式列為進階，視需要再深入 |
