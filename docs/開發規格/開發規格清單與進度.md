## 階段 2 目標

定義「RM300P UHF RFID Reader bench tool」的開發規格書。工具用途：拿到設備後，
驗證序列通訊與封包協定是否正確、實測標籤盤點的觸發方式（單次／連續開／連續關），
作為後續（階段 3）實作與（階段 4/5）測試驗收的依據。

- **技術棧**：C# / .NET 8 以上、Console（光棒選單導覽）、`System.IO.Ports.SerialPort`。
- **第一版範圍排除**：Kill(0x85)、Lock(0x84)、BlockPermalock(0x87)、韌體更新(8.1~8.3)
  等不可逆／可能變磚的操作（文件保留，工具第一版不實作）。

## 結案備註

- 審閱結論：規格書無需修改，內容通過。
- **測試策略調整（2026-06-16 使用者拍板）**：目前尚未取得 RM300P 實體設備，
  因此測試順序調整為**先完成階段 3 實作（含離線單元測試），待設備到位後再補實機測試（階段 4）**。
  - 離線可完成：協定/分包/CHECKSUM 單元測試（04 階段 -1）、Console UI、全部指令 API 封裝。
  - 待設備才能做：階段 0~6 實機確認測試（連線、握手、盤點三模式實測等）。

## 已鎖定的關鍵決策（來源：`.claude/daily/2026-06-13/`）

1. 技術棧 = C#/.NET 8+；第一版工具形態 = 互動式 Console bench tool；階段 2 先產出開發規格文件。
2. UI = **光棒（highlight bar）選單導覽**，階層為「分類（Category）> 功能／設定」，方向鍵移動、Enter 進入/執行。
3. 工具須能實測**盤點觸發三模式**：單一觸發收集／開啟連續收集／關閉連續收集，並即時呈現標籤與原始封包。
4. 排除不可逆/變磚操作（Kill/Lock/BlockPermalock/韌體更新）。
